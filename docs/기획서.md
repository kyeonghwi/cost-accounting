## 2. 추진 배경 및 핵심 문제

### 배경

원가/관리회계는 기업이 간접비를 어떻게 제품·프로젝트·부서에 배부하고, 실제 성과와 예산의 차이를 어떻게 분석하는지를 다루는 핵심 회계 영역이다. 보험사·자산운용사 등 금융기관에서는 원가 데이터를 기반으로 사업부별 수익성 분석, 이전가격 산정, 예산 편성을 수행한다.

### 핵심 문제

복잡한 원가회계 계산 로직(배부율 계산, 차이 분해, 이전가격 적용)을 코드로 정확하게 구현하는 것은 어렵다. 특히 두 가지 문제가 있다:

1. **금액 정밀도**: JavaScript의 `number` 타입(IEEE 754 배정밀도)은 소수점 계산에서 오류가 발생한다. 금융 데이터에서 `0.1 + 0.2 = 0.30000000000000004` 수준의 오류는 허용되지 않는다.
2. **검증 어려움**: 알고리즘이 복잡할수록 버그를 숨기기 쉽다. 수동 계산으로 검증하지 않으면 코드가 잘못된 결과를 조용히 반환한다.

## 3. 개발 목표

- 원가/관리회계의 핵심 계산 알고리즘(배부, 차이분석, 이전가격)을 **정확하게** 코드로 구현한다

- 회계 담당자·프로젝트 관리자·경영진 등 역할별로 맞춤 대시보드를 제공한다

## 4. 핵심 기능 정의

### 기능 1. 원가배부 엔진 (Allocation Engine)

간접비 원가집합(Cost Pool)의 잔액을 원가 대상(부서·프로젝트)에 배부하는 엔진.

**두 가지 배부 방법 지원:**

| 방법 | 설명 | 특징 |
|------|------|------|
| **직접배부법** (Direct) | 서비스 부서 원가를 생산 부서에 직접 배부 | 부서 간 서비스 제공 무시 |
| **단계배부법** (Step-Down) | 서비스 부서를 순서대로 배부, 이미 배부된 부서는 수령 불가 | 부분적 부서 간 관계 반영 |

**배부 드라이버:** 인원 수(HEADCOUNT), 직접 노동시간(DIRECT_LABOR_HOURS), 직접 원가(DIRECT_COST), 사용자 정의 수치

**정확성 보장:**
- 배부 후 `Σ 배부액 = 풀 잔액`이 항상 성립하도록 마지막 단계 잔차 조정
- SHA-256 체크섬으로 배부 입력·결과를 감사 기록 (`AllocationRun`)
- 잔차가 풀 잔액의 0.5%를 초과하면 오류 발생

### 기능 2. 표준원가 차이분석 엔진 (Variance Decomposition)

실제 결과와 예산의 차이를 4개 요인으로 분해하는 Kaplan/Atkinson 4분법 적용.

| 차이 요인 | 수식 | 의미 |
|-----------|------|------|
| **가격 차이** | (실제단가 - 표준단가) × 실제수량 | 가격 결정의 성과 |
| **수량 차이** | (실제수량 - 예산수량) × 표준단가 | 판매량의 성과 |
| **믹스 차이** | (실제믹스 - 예산믹스) × 실제수량 × 표준단가 | 제품 구성 변화의 영향 |
| **효율 차이** | (표준단가 - 실제단가) × 실제수량 | 원가 효율성의 성과 |

4개 요인의 합 = 총 차이 (잔차 0). 긍정값 = 유리한 차이, 부정값 = 불리한 차이.

### 기능 3. 이전가격 엔진 (Transfer Pricing)

기업 내 조직 간 서비스·자원 이전 시 적용하는 가격 계산 엔진.

- 표준 요율(StandardRate) 기반 이전가격 계산
- 마크업(TransferMarkup) 적용으로 조직별 수익성 반영
- 이전 공급 조직: 이전 수익(Transfer Credit) 발생
- 이전 수령 조직: 이전 비용(Transfer Charge) 발생

### 기능 4. 공헌이익 분석 (Contribution Margin)

CVP(원가-조업도-이익) 분석으로 제품·서비스의 수익성 파악.

- 공헌이익 = 매출 - 변동원가
- 공헌이익률, 손익분기점 계산

### 기능 5. 월 마감 워크플로우 (Monthly Close)

월별 원가 계산을 하나의 데이터베이스 트랜잭션으로 실행.

```
월 마감 실행 (runClose)
    │
    ├─ 1. 기간(Period) 상태 검증 (OPEN인지 확인)
    ├─ 2. 배부 규칙 적용 → AllocationResult 생성
    ├─ 3. 이전가격 적용 → TransferEntry 생성
    ├─ 4. 차이분석 스냅샷 생성 → VarianceSnapshot
    └─ 5. 기간 상태 OPEN → CLOSED 변경
         (모든 작업이 단일 $transaction 내에서 실행)
```

### 기능 6. 역할별 대시보드

| 역할 | 대시보드 | 주요 지표 |
|------|----------|-----------|
| 경영진 (Executive) | 전사 손익 통합 | HQ별 공헌이익, 전사 P&L |
| 관리회계 담당자 (Controller) | HQ 드릴다운 | 직접원가, 배부원가, 이전가격 |
| 프로젝트 관리자 (PM) | 프로젝트 예실 대비 | 예산 vs 실제 vs 표준 |
| 원가담당 (Cost Accountant) | 차이분석 | 4분법 차이 분해 막대 차트 |

## 8. 데모 시나리오

**시드 데이터 구성:**
- 1 본사 (HQ), 2개 사업부 (기계, 조립), 공유 서비스 원가집합 3개 (HR, IT, 시설)
- 3개 작업지시서 (표준원가 vs 실제원가 추적)
- 2개 기간의 차이 보고서

**역할 전환 데모:**
페르소나 선택기에서 Executive → Controller → PM → Cost Accountant 순서로 전환하며 동일 데이터를 각 역할에 맞는 관점에서 확인.

**월 마감 데모:**
1. Close 메뉴 → OPEN 기간 선택 → "마감 실행" 클릭
2. 배부 → 이전가격 → 차이분석 스냅샷이 단일 트랜잭션으로 완료
3. Variance Dashboard에서 4분법 차이 분해 막대 차트 확인
