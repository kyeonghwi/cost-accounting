# 배분 알고리즘

## 개요

배분 엔진은 간접원가 풀의 잔액을 원가 대상(작업, 제품, 부서)에 배부한다. 배부 기준(드라이버) 방식을 사용하며, 직접 배분법과 단계 배분법(순차) 두 가지를 지원한다.

**소스 파일**: `lib/engines/allocation.ts`

---

## 직접 배분법

직접 배분법은 각 서비스 부서의 원가를 제조 부서에 직접 배분한다. 서비스 부서 간의 상호 서비스는 무시한다.

### 배분 기준

| 기준 | 설명 | 드라이버 근거 |
|------|------|--------------|
| `HEADCOUNT` | 수혜 부서의 종업원 수 | 기간 초 인원 수 |
| `DIRECT_LABOR_HOURS` | 수혜 부서의 직접 노동 시간 | 기간 실제 직접노무시간(DLH) |
| `DIRECT_COST` | 수혜 부서의 직접재료비 + 직접노무비 | 기간 실제 원가 |

풀의 `customKey` 필드를 통해 사용자 정의 수치 드라이버도 지원한다.

### 배분 공식

잔액 `B`인 풀과 수혜 원가 대상 집합에 대해:

```
drierTotal  = Σ driver[i]   (모든 수혜 대상 i)
rate[i]      = driver[i] / driverTotal
allocated[i] = B × rate[i]
```

## 단계 배분법

단계 배분법(순차 배분)은 서비스 부서 간 부분적 상호 관계를 인식한다. 부서를 우선순위 점수(예: 총 원가 또는 경영진 판단)로 순위를 매긴다. 각 부서는 잔액을 한 번만 배분하고 이후 추가 배분을 받지 않는다.

### 공식

단계 배분법은 직접 배분법과 동일한 비율 공식을 적용하되, 수혜 집합에서 이전 단계에 이미 마감된 부서를 제외한다.

```
단계 k (부서 D_k가 배분하는 경우):
  receivingSet = { 모든 원가 대상 } \ { D_1, ..., D_{k-1} }
  rate[i]      = driver[i] / Σ driver[j] (j ∈ receivingSet)
  allocated[i] = balance(D_k) × rate[i]
```

## 반올림 조정

비율이 균등하게 나누어지지 않을 경우 부동소수점 연산에서 잔차가 발생할 수 있다. 엔진은 최종 단계에서 잔차 조정을 적용한다:

```
residual = B - Σ allocated[i]   (마지막 수혜 대상을 제외한 모든 대상)
allocated[last] += residual
```

이 처리로 부동소수점 오류에 관계없이 `Σ allocated[i] = B`가 정확히 보장된다.

조정 범위에 제한이 있다: `|residual| > 0.005 × B`이면 엔진이 오류를 발생시켜 대규모 불일치를 묵인하지 않는다.
