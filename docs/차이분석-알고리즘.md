# 차이분석 알고리즘 — Kaplan/Atkinson 4요인 분해

## 개요

차이분석 엔진은 Kaplan/Atkinson 프레임워크에 따라 총 표준원가 차이를 네 가지 가산 효과로 분해한다. 각 효과는 실제 결과와 정적(마스터) 예산 사이의 격차 원인 하나를 분리한다. 구현체는 `lib/variance/decompose.ts`에 있다.

---

## 정의

| 기호 | 의미 |
|------|------|
| `AP` | 실제 단위 가격 (단위당 수익) |
| `SP` | 표준 단위 가격 (예산 가격) |
| `AV` | 실제 판매량 (판매 단위 / 청구 시간) |
| `BV` | 예산 판매량 |
| `AM` | 실제 판매 구성비 (0..1; 단일 제품이면 1.0) |
| `BM` | 예산 판매 구성비 (0..1; 단일 제품이면 1.0) |
| `AC` | 실제 단위 원가 |
| `SC` | 표준 단위 원가 (단위당 예산 원가) |

---

## 공식

### 가격 효과 (Price Effect)

실제 판매량에 대해 표준과 다른 가격을 적용했을 때의 수익 영향을 측정한다.

```
priceEffect = (AP − SP) × AV
```

AP > SP이면 유리한 차이 (양수 값).

### 물량 효과 (Volume Effect)

표준 단위 가격 기준으로 예산 대비 다른 총 판매량을 달성했을 때의 수익 영향을 측정한다.

```
volumeEffect = (AV − BV) × SP
```

실제 판매량이 예산을 초과하면 유리한 차이 (양수 값).

### 구성비 효과 (Mix Effect)

실제 제품 구성비가 예산 구성비와 달라짐으로써 발생하는 원가/이익 영향을 측정한다. 단일 제품의 경우 AM과 BM이 모두 1.0이므로 mixEffect = 0.

```
mixEffect = (AM − BM) × AV × SP
```

실제 구성비가 고가 제품 쪽으로 이동하면 유리한 차이.

### 능률 효과 (Efficiency Effect)

실제 생산량에 대해 표준 허용량보다 더 많거나 적은 자원을 소비했을 때의 원가 영향을 측정한다.

```
efficiencyEffect = (SC − AC) × AV
```

단위당 실제 원가가 표준보다 낮으면 유리한 차이 (양수 값).

---

## 총 차이와 잔차

이 선형 모델에서 네 가지 효과의 합계는 총 차이와 정확히 일치한다. 가격 × 물량 등 교차항이 포함되지 않으므로 잔차는 분석적으로 0이다.

```
totalVariance = priceEffect + volumeEffect + mixEffect + efficiencyEffect
residual      = 0
```

조정 제약 조건:

```
|residual| / |totalVariance| ≤ 0.001   (0.1%)
```

`totalVariance = 0`인 경우, 엔진은 나눗셈 대신 `residual = 0`을 정확히 검증한다.

---

## 부호 규칙

모든 효과는 유리한 차이를 양수로 표현한다:

- 양수 값 → 유리 (실제가 표준/예산보다 우수)
- 음수 값 → 불리 (실제가 표준/예산보다 열위)

---

## 계산 예시

입력값:

| 항목 | 예산 / 표준 | 실제 |
|------|------------|------|
| 판매량 | 100 | 120 |
| 단위 가격 | 100 | 110 |
| 단위 원가 | 60 | 70 |
| 구성비 | 0.50 | 0.60 |

계산:

```
priceEffect      = (110 − 100) × 120 = 1,200
volumeEffect     = (120 − 100) × 100 = 2,000
mixEffect        = (0.60 − 0.50) × 120 × 100 = 1,200
efficiencyEffect = (60 − 70) × 120 = −1,200

totalVariance = 1,200 + 2,000 + 1,200 + (−1,200) = 3,200
residual      = 0
```

잔차 검증: 0 / |3,200| = 0 ≤ 0.001 ✓

## 출력 형태

```ts
interface VarianceComponents {
  priceEffect:      Decimal  // 금액 — toString()은 소수점 이하 4자리 반환
  volumeEffect:     Decimal  // 금액
  mixEffect:        Decimal  // 금액
  efficiencyEffect: Decimal  // 금액
  totalVariance:    Decimal  // 금액
  residual:         Decimal  // 금액 — 선형 모델에서 항상 0
}
```

모든 Decimal 값은 `MONEY_SCALE = 4` 소수점으로 직렬화된 `Money` 인스턴스(`lib/allocation/decimal.ts`)다.
