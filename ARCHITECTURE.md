## Key Design Decisions

### Decimal Arithmetic
All monetary computations use Prisma `Decimal` end-to-end. `lib/allocation/decimal.ts` configures `Decimal.precision=28` and patches `Decimal.prototype.plus` to propagate `Money` subclass. A double-patch guard prevents re-entry on multiple imports.

### Checksum Strategy
`lib/allocation/checksum.ts` produces deterministic SHA-256 input/output checksums stored in `AllocationRun`. Canonical sort key: `(fromPoolOrgId, toProjectId)`. Stored digests break if sort order or serialization changes — marked `@AX:ANCHOR`.

### ConservativeAdd
`conservativeAdd` in `decimal.ts` guarantees `sum(allocated) === pool` exactly via last-step rounding adjustment. All allocation callers depend on this post-condition for REQ-ALLOC-01 conservation.
